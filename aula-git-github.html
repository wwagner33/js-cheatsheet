Git & GitHub Essencial

O Guia Prático de 30 Minutos

O que é Git?

É um Sistema de Controle de Versão. Pense nele como "pontos de salvamento" (save points) superpoderosos para o seu código.

Analogia: O histórico de versões do Google Docs, mas no seu computador e para qualquer tipo de arquivo.

O que é GitHub?

É uma plataforma online para hospedar seus repositórios Git.

Analogia: Se o Git é o software que escreve o livro, o GitHub é a biblioteca online onde você guarda, compartilha e colabora nos seus livros.

Configuração Inicial (Faça uma vez)

Antes de tudo, apresente-se ao Git. Abra seu terminal e digite:

git config --global user.name "Seu Nome"
git config --global user.email "seu.email@exemplo.com"

Isso identifica você em todos os seus commits.

Passo 1: Iniciando um Repositório

Para começar a versionar um projeto, use o comando git init.

# Crie uma pasta para o projeto
mkdir meu-projeto
cd meu-projeto

# Inicie o Git nela
git init

Pronto! Uma pasta oculta .git foi criada para guardar todo o histórico.

O Ciclo Básico: Add & Commit

O fluxo de trabalho local tem 2 etapas principais:

  1. git add: Adiciona arquivos na "área de preparação" (Staging Area).
  2. git commit: Salva permanentemente as mudanças que estão na Staging Area.

Analogia: `git add` é colocar as compras na esteira do caixa. `git commit` é pagar e levar o recibo.

Exemplo Prático: Primeiro Commit

Dentro da pasta meu-projeto, vamos criar e salvar um arquivo.

# Crie um arquivo HTML simples
echo "<h1>Meu Site</h1>" > index.html

# Adicione o arquivo à Staging Area
git add index.html

# Salve o commit com uma mensagem descritiva
git commit -m "Cria a página inicial do projeto"

Comandos Úteis: Status e Log

Como saber o que está acontecendo?

git status: Mostra quais arquivos foram modificados e o que está na Staging Area. Use o tempo todo!

git status

git log: Mostra o histórico de todos os commits feitos.

git log --oneline

Passo 2: Conectando com o GitHub

Vamos enviar nosso projeto para o mundo!

  1. Vá para GitHub.com e crie um novo repositório (sem README ou .gitignore).
  2. O GitHub te dará a URL do repositório. Use-a nos comandos abaixo.

Isso conecta seu repositório local a um "controle remoto" chamado origin.

Enviando o Código: git push

Execute estes comandos no seu terminal, substituindo a URL.

# Conecta seu repositório local ao remoto (faça isso uma vez)
git remote add origin https://github.com/seu-usuario/meu-projeto.git

# (Opcional) Renomeia sua branch principal para 'main'
git branch -M main

# Envia seus commits para o GitHub
git push -u origin main

Atualize a página do GitHub. Seus arquivos estarão lá!

O Ciclo do Dia a Dia

O fluxo de trabalho diário, sozinho ou em equipe, é:

  • git pull: Baixar as últimas alterações do GitHub.
  • ... Fazer suas modificações no código ...
  • git add: Adicionar seus arquivos modificados.
  • git commit: Salvar suas alterações localmente.
  • git push: Enviar suas alterações para o GitHub.

Exemplo Prático: Recebendo Mudanças

Vamos simular o trabalho de um colega.

  1. No site do GitHub, abra o arquivo `index.html` e edite-o (adicione um parágrafo, por exemplo). Salve a alteração.
  2. Agora, no seu terminal, rode o comando abaixo para "puxar" a alteração:
git pull origin main

Abra o arquivo `index.html` no seu computador. A mágica aconteceu!

O Poder dos Branches (Ramos)

Branches são "universos paralelos" do seu projeto.

Eles permitem que você trabalhe em novas funcionalidades (ex: `feature-login`) sem quebrar a versão principal (`main`).

# Cria um novo branch e muda para ele
git checkout -b nova-feature

# Para voltar ao branch principal
git checkout main

Resumo Essencial

  • git init: Inicia um repositório.
  • git status: Verifica o estado atual.
  • git add [arquivo]: Adiciona para a próxima "foto".
  • git commit -m "mensagem": Tira a "foto" (salva).
  • git push: Envia suas "fotos" para o GitHub.
  • git pull: Puxa as "fotos" mais recentes do GitHub.
  • git checkout -b [branch]: Cria e muda para um novo ramo.
